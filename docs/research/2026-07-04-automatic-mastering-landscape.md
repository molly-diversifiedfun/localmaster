**The algorithm (spec-level, from source review — this section can serve as the written spec):**

Pipeline is 4 sequential stages (matchering/stages.py — https://github.com/sergree/matchering/blob/master/matchering/stages.py):

1. **Level match** (`__match_levels`): normalize reference; split both tracks to mid/side; divide into pieces ≤15 s at 44.1 kHz; per-piece RMS; "loudest pieces" = pieces with RMS ≥ average RMS; match statistic = RMS over loudest pieces only; gain = ref_match_rms / max(ε, target_match_rms) applied to mid and side (https://github.com/sergree/matchering/blob/master/matchering/stage_helpers/match_levels.py). Loudest-pieces-only makes it robust to intros/breakdowns. It uses plain RMS, not BS.1770 — **LocalMaster can substitute its existing gated BS.1770 loudness and arguably improve on the original.**
2. **Frequency match** (`__match_frequencies`): averaged rectangular-window magnitude spectra of the loudest pieces (scipy.signal.stft, boxcar, nperseg=4096, noverlap=0); matching curve = ref_avg_fft / max(target_avg_fft, 1e-6); interpolate to a log-frequency grid (4× oversampling), smooth with LOWESS (frac=0.0375), interpolate back; FIR = irfft(curve) with a Hann window (4096-tap linear-phase); apply via fftconvolve separately on mid and side (https://github.com/sergree/matchering/blob/master/matchering/stage_helpers/match_frequencies.py). **100% numpy/scipy/statsmodels — the easiest reimplementation of anything surveyed.** Only judgment calls: LOWESS bandwidth and log-grid interpolation.
3. **Level correction** (`__correct_levels`): iterative post-EQ refinement — clip, compare local RMS to reference, corrective gain, repeat (default 4 steps).
4. **Finalize**: optional brickwall limiter, optional normalize, multiple output variants.

M/S decomposition persists through stages 1–2, which is how stereo width gets matched (side channel EQ'd and gained independently).

**Strength slider (0–100%):** natural insertion points fall out of the spec — interpolate the matching EQ curve between unity and the full ratio in log-magnitude domain (`curve^s` or `s·curve_dB`), and interpolate the level-match gain in dB. This is a LocalMaster design decision, not something in matchering, so no license concern.

## 2. Style/genre-adaptive chains (the LANDR problem)

- **LANDR patent US 9,304,988** (https://patents.google.com/patent/US9304988B2/en): extracts "chromosomal" numeric features (tempo, MFCCs, sub-band flux), classification features (genre, instrumentation), and production features (spectral shape, dynamic range, loudness, stereo width, masking); genre queries a reference-record production database; matched records supply target production features; an interpreter sets chain order and a mapper sets processor parameters. Architecture-level, useful as a design pattern.
- **LANDR public description** (https://www.landr.com/what-is-ai-mastering-and-how-does-it-work, corroborated by https://www.soundonsound.com/techniques/landr-cloudbounce-future-mastering): micro-genre detection → scan dynamics/frequency balance/stereo width → chain of multiband compression, EQ, stereo enhancement, limiting, saturation. Marketing-level only — not reimplementable from public sources.
- **Ozone 9 Master Assistant published heuristics** (https://s3.amazonaws.com/izotopedownloads/docs/ozone9/en/master-assistant/index.html) — the best concrete rule set found: **crest factor as the dynamics gate ("ideal range: 3–10")**, light low-end compression only when crest factor exceeds thresholds; maximizer set from a LUFS target (Low −14 / Medium −12 / High −11 integrated) and destination true-peak ceiling (Streaming −1.0 dB, CD −0.3 dB); dynamic EQ nodes placed on peak frequencies identified during limiting; analysis needs ~30 s of the loudest section.

**Investigated and refuted — do not rely on:** the claim that Ozone's genre mechanism is a pure heuristic similarity-scorer against ten corpus-derived genre curves. The ten-genre-curve **template-blend architecture is real** (iZotope's own docs confirm per-genre similarity percentages linearly blended into a unique target curve, EQ moved toward it), but the similarity weights come from a **trained neural-net classifier at inference**, per iZotope's Ozone 8 docs — not from heuristic spectral-distance scoring. Takeaway: LocalMaster can adopt the template-blend architecture (static target curves, linear blend, EQ-toward-target) but must substitute its own heuristic similarity scorer (e.g., log-spectral distance from the input's average spectrum to each genre curve) — a sound design, but not a faithful replication of iZotope's runtime.

## 3. Automatic multiband compression / parameter estimation

All three key papers are equation-complete for clean-room numpy reimplementation — no code adoption needed, so no license issues at all in this area.

1. **Giannoulis/Massberg/Reiss, "Digital Dynamic Range Compressor Design — A Tutorial and Analysis," JAES 60(6):399–408, 2012** (AES e-lib ID 16354, https://aes.org/publications/elibrary-page/?id=16354; open PDF via https://web.archive.org/web/2023/https://www.eecs.qmul.ac.uk/~josh/documents/2012/GiannoulisMassbergReiss-dynamicrangecompression-JAES2012.pdf). Verified complete: hard-knee curve y_G = T+(x_G−T)/R; quadratic soft knee; one-pole coefficient α = exp(−1/(τ·fs)); smooth-branching peak detector; feedforward control voltage. Recommended topology per the paper's own conclusion: **feedforward, log-domain level detection placed after the gain computer, smooth-branching one-pole detector, soft knee.** Verdict: REIMPLEMENTABLE-CLEAN-ROOM, rank 1. Run per-band instances behind a Linkwitz-Riley crossover (textbook scipy).
2. **Giannoulis/Massberg/Reiss, "Parameter Automation in a Dynamic Range Compressor," JAES 61(10):716–726, 2013** (https://aes.org/e-lib/browse.cfm?elib=16965; open PDF via Wayback of the QMUL author page). Verified: latency-free crest-factor estimator (peak² and RMS² one-poles, shared τ=200 ms); **auto ballistics τ_A = 2·80ms/crest², τ_R = 2·1s/crest² − τ_A**; auto-knee W = 2.5·(c_dev+c_est) with ratio fixed at ∞; auto make-up gain = −(c_dev+c_est) or EBU R128 loudness matching. Threshold stays manual in this paper. Verdict: REIMPLEMENTABLE-CLEAN-ROOM, rank 1 for automation.
3. **Ma/De Man/Pestana/Black/Reiss, "Intelligent Multitrack Dynamic Range Compression," JAES 63(6):412–426, 2015** (open PDF: https://www.eecs.qmul.ac.uk/~josh/documents/2015/Ma%202015%20-%20Intelligent%20Multitrack%20Compression.pdf; index https://www.aes.org/e-lib/browse.cfm?elib=17821). Verified regression rules: **R = 0.54·w_p + 0.764·w_f + 1; T = −11.03 + 0.44·x_RMS − 4.987·w_p (dB); knee W = |T|/2**; ballistics identical to the 2013 paper; make-up gain via EBU R128 loudness match. Caveat: w_p/w_f are cross-adaptive (defined relative to other stems), so single-file mastering use requires substituting a fixed reference crest factor — a documented adaptation, not in the paper. Verdict: REIMPLEMENTABLE-CLEAN-ROOM with adaptation, rank 2.
4. **Maddams/Finn/Reiss, "An Autonomous Method for Multi-Track Dynamic Range Compression," DAFx-12** (https://www.dafx12.york.ac.uk/papers/dafx12_submission_6.pdf). Simplest verified auto-threshold rule: **threshold = RMS − 12 dB** (knee 3 dB) in ratio mode; compression amount driven by EBU Tech-3342 **LRA (10th–95th percentile of short-term loudness)** — more compression for high-LRA material; make-up gain matches integrated loudness in/out. Verdict: REIMPLEMENTABLE-CLEAN-ROOM, easiest entry point.

## 6. Flags and gaps

- **No VERIFIED claim carried an "UNVERIFIED" source** — all findings above trace to fetched primary sources (repo files, PDFs, patent, vendor docs).
- **Refuted claim 1 (Ozone heuristic-similarity):** partially wrong as stated; template-blend is real, similarity scoring is a neural net — see Section 2.
- **Refuted claim 2 (chardet AI-rewrite precedent):** the 2026 chardet LGPL→MIT rewrite is a **live dispute over an explicitly non-clean-room rewrite** (Pilgrim contests the relicense; JPlag figures were misquoted; Blanchard conceded no clean-room separation). It is a cautionary tale, not legal cover. A true clean-room needs a separate spec-writer and implementer — one session reading GPL code doesn't qualify. The spec in Section 1 of this report exists precisely to enable that separation.
- **Refuted claim 3 (Matchering dependency audit):** its deps (numpy, scipy, soundfile, resampy, statsmodels) are indeed all permissive (BSD-3/ISC), but numpy's BSD-3 status comes from its LICENSE.txt, not the GitHub license API (which returns NOASSERTION), and pyloudnorm (MIT) is not a matchering dependency at all. The permissive-deps conclusion survives; the claimed verification method did not. Irrelevant to LocalMaster anyway since matchering itself (GPL-3.0) won't be imported.
- **Coverage gaps** (angles investigated but yielding no surviving verified claims): licenses for DeepAFx-ST, ITO-Master, DiffMST, phaselimiter, pedalboard, Calf/LSP/Zam were not license-verified in the surviving findings — verify each LICENSE file before touching any of them; treat pedalboard as suspected GPL-3.0 and Calf/LSP/Zam as suspected GPL until confirmed. Listening-test evidence on DSP-vs-human quality ceilings also did not survive verification — the guardrails implicit in the verified sources (EQ-curve smoothing, loudest-section analysis, LUFS/TP targets) are the only supported ones.
